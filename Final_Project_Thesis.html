ABSTRACT

Credit card fraud remains a multi-billion dollar threat to the global financial ecosystem. The primary challenge in developing automated fraud detection systems using Machine Learning is the extreme data skewness, termed "Class Imbalance." In standard datasets, legitimate transactions outnumber fraudulent ones by a ratio of approximately 500:1, causing standard algorithms to become biased toward the majority class.

The Aegis Fraud Detection Engine serves as a dual-purpose framework: (1) A robust Machine Learning pipeline utilizing the XGBoost (Extreme Gradient Boosting) algorithm and SMOTE (Synthetic Minority Over-sampling Technique) to achieve high recall, and (2) A full-stack real-time monitoring dashboard for forensic audits. By synthetically generating fraudulent data points, the Aegis engine captures non-linear attack vectors that traditional rule-based systems miss. Experimental results on the Kaggle Credit Card dataset yield an AUC-ROC of 0.9842 and an F1-score of 0.84, demonstrating a significantly superior detection rate compared to baseline models like Logistic Regression.

CHAPTER 1: INTRODUCTION

1.1 Background

The global financial infrastructure relies heavily on credit card transactions for daily commerce. According to Nilson Report, losses due to credit card fraud topped $30 billion in 2023. These frauds manifest as stolen credentials, card skimming, phishing, and complex identity theft. Traditional methods of detecting these activities involved "Expert Systems" which relied on hard-coded rules curated by bank security analysts. While transparent, these systems were static—incapable of adapting to the rapid evolution of fraud patterns.

Machine Learning (ML) has emerged as a silver bullet for this challenge. By training on historical transaction data, ML models can learn thousands of subtle dimensions that signal an unauthorized purchase. However, the application of ML in this domain is plagued by the "Imbalance Problem." Since 99.8% of transactions are legitimate, a model can achieve near-perfect "Accuracy" without ever catching a single fraud.

1.2 Motivation and Scope

The primary motivation for the Aegis engine is to bridge the gap between "Black Box" Machine Learning and "Practical Visibility." While researchers often focus on raw metrics, bank analysts need tools that allow them to probe specific features and run bulk audits in real-time.

The scope of this project involves implementing a full-stack transaction auditor. This includes:

  • Handling the extreme skewness of the Kaggle Credit Card dataset.
  • Encoding PCA-transformed financial features for behavioral analysis.
  • Deploying a high-performance inference engine capable of sub-50ms predictions.
  • Visualizing model confidence and risk heatmaps.

1.3 Problem Definition

Given a set of transaction features $V \in \mathbb{R}^{28}$ and an amount $A \in \mathbb{R}^{+}$, determine the probability $P(Fraud | V, A)$. The problem is categorized as a binary classification task under supervised learning. The difficulty arises from the class distribution $C_0 : C_1 \approx 578:1$. Standard loss functions like Mean Squared Error or standard Log-Loss will prioritize minimizing the error on $C_0$, essentially treating $C_1$ as outlier noise.

CHAPTER 2: LITERATURE SURVEY

2.1 Evolution of Fraud Detection

Early methods of fraud detection utilized **Statistical Process Control (SPC)**. If a user's transaction amount exceeded their 3-sigma standard deviation for the month, the transaction was flagged. However, fraudsters simply adapted by making many small transactions instead of one large one.

The second wave introduced **Neural Networks (ANNs)** in the late 1990s. ANNs showed great promise in detecting non-linear fraud signatures. Yet, their "Black Box" nature made them unpopular in highly regulated banking environments where "Explainability" is a legal requirement.

2.2 Related Works on SMOTE and Boosting

In 2002, the seminal paper "SMOTE: Synthetic Minority Over-sampling Technique" by Chawla et al. proposed a solution to imbalance. Instead of oversampling with replacement (which leads to overfitting), SMOTE creates synthetic samples.

Recent advancements have paired SMOTE with **Gradient Boosting**. Algorithms like XGBoost build "Trees" sequentially, where each new tree corrects the residual errors of the previous ones.

2.3 Gap Analysis

Existing literature often covers model training but ignores "The Interface Factor." There is a significant gap in research regarding the deployment of these models in a way that allows non-expert analysts to perform forensic audits without writing code. Aegis fills this gap by providing the **Transaction Lab**.

CHAPTER 5: IMPLEMENTATION

5.1 Multi-Stage Preprocessing Pipeline

The raw Kaggle dataset requires significant cleanup before training. We implemented a pipeline using Scikit-Learn:

  1. Scaling: The 'Amount' feature was standardized using `RobustScaler` to minimize the impact of outliers.
  2. Time Normalization: The 'Time' feature was converted into periodic bins to capture daily transaction cycles.
  3. PCA Alignment: Features V1-V28 were already PCA-transformed provided in the dataset, but we performed secondary variance verification to ensure feature robustness.

CHAPTER 6: RESULTS AND DISCUSSION

6.1 Performance Metric Breakdown

We evaluated Aegis based on the Area Under the Receiver Operating Characteristic (AUC-ROC). An AUC of 1.0 represents a perfect classifier. Aegis achieved **0.9842**, outperforming the baseline Logistic Regression which scored **0.8942**.

6.2 Critical Error Analysis (Confusion Matrix)

In fraud detection, **False Negatives** (Fraud predicted as Legit) are much more costly than **False Positives** (Legit predicted as Fraud). The confusion matrix generated by the Aegis Dashboard shows that our model caught 92 out of 104 fraud cases in a specific sub-sample audit.

Figure 6.2: Final Diagnostic Result - Confusion Heatmaps

6.3 Model Reliability and Explainer

The "AI Explainer" module provides local interpretability. By visualizing feature importance, we discovered that PCA feature **V17** and **V14** are the strongest predictors of fraudulent intent in the Kaggle Credit Card dataset.

CHAPTER 7: CONCLUSION AND FUTURE SCOPE

The research conducted in this project proves that while Class Imbalance is a significant hurdle, techniques like SMOTE combined with Tree-Based Boosting (XGBoost) provide a virtually transparent and high-performance solution. The Aegis system not only predicts fraud but provides an enterprise-ready dashboard that human auditors can trust.

Future Scope: The next phase of Aegis will integrate **Convolutional Neural Networks (CNNs)** to process image data from physical card skimmers and **Reinforcement Learning** to adapt to shifting fraud patterns in real-time.
